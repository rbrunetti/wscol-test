Examples

WSCoL is a very flexible language and for this reason it could be used in various contexts. In the following sections you’ll its application.

Toy example: Book store

Just a little case to understand better about what we’re talking about. The input file is the succeeding xml file, book.xml

<inventory>
    <name>InventoryABCD</name>
    <book>
        <year>2000</year>
        <title>Snow Crash</title>
        <authors>
            <author>Neal Stephenson</author>
        </authors>
        <publisher>Spectra</publisher>
        <isbn>0553380958</isbn>
        <price>15</price>
    </book>
    <book>
        <year>2005</year>
        <title>Burning Tower</title>
        <authors>
            <author>Larry Niven</author>
            <author>Jerry Pournelle</author>
        </authors>
        <publisher>Pocket</publisher>
        <isbn>0743416910</isbn>
        <price>6</price>
    </book>
    <book>
        <year>1995</year>
        <title>Zodiac</title>
        <authors>
            <author>Neal Stephenson</author>
        </authors>
        <publisher>Spectra</publisher>
        <isbn></isbn>
        <price>7.50</price>
    </book>
</inventory>

Now we have to generate a wscol file with the assertions to verify, save assertions.wscol in the same folder and copy the assertion.

/inventory/book[1]/title == "Snow Crash";

Launch the application from cli with the command

java –jar WSCoL-App.jar assertion.wscol –xml book.xml

The assertion says that the title of the first book (note the use of the predicate in square brackets) is equal to “Snow Crash” and, obviously, the result is true. The expression returns a String and correctly compare it with another string on the right side.
